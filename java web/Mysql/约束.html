# 约束

介绍：约束是作用于表中字段上的规则，用于限制存储在表中的数据

目的：保证数据库中数据的正确、有效性和完整性

约束的分类如下

约束 描述
非空约束not null 限制该字段的数据不能为null，必须有值
唯一约束unique 保证该字段的所有数据都是唯一的，也就是不能重复
主键约束primary key 主键是一行数据的唯一标识，要求不能为null，且必须唯一
默认约束default 保存数据的时候如果没有自动该字段的值，则采用默认值
检查约束(8.0.16版本后)check 保证字段满足某一个条件
外键约束foreign key 用来保证两张表的的数据之间建立连接，保证数据的一致性和完整性

注意：约束是作用于表中字段上的，可以在创建表/修改表的时候添加约束

添加外键约束

constraint 外键名 foreign key (外键字段名) references 主表表名(主键字段名)

# 使用约束

案例需求： 根据需求，完成表user结构的创建。需求如下：

字段 含义 类型 约束套件 约束关键字
id 唯一标识 int 主键且自动增长 primary key, auto_increment
name 姓名 varchar(10) 不为空且唯一 not null, unique
age 年龄 int 0~120 check
status 状态 char(1) 如果没有指定值，默认为1 default
gender 性别 char(1) 无 
create table user
(
    id     int auto_increment primary key comment '唯一标识',
    name   varchar(10) not null unique comment '姓名',
    age    int check ( age between 0 and 120) comment '年龄',
    status char(1) default '1' comment '状态',
    gender char(1) comment '性别'
);

# 外键约束

外键：用来让两张表的数据之间建立连接，从而保证数据的一致性和完整性。

注意：目前上述两张表，只是在逻辑上存在这样一层关系；在数据库层面，并未建立外键关联， 所以是无法保证数据的一致性和完整性的。

# 添加外键

建表的时候添加外键约束

create table emp
(
    id     int primary key auto_increment,
    name   varchar(20),
    age    int,
    dep_id int,
    constraint fk_emp_dept_01 foreign key (dep_id) references dept (id)
);

注意：

外键约束语法

constraint 外键名 foreign key (外键字段名) references 主表表名(主键字段名)
1

对于有以下一个外键约束的例子

alter table table1
    add constraint fk_t1_t2 foreign key (tid1) references table2 (tid2);
1
2

解释：

  • table1是子表，table2是主表
  • table2的tid2是table2的主键或唯一约束列
  • table1的tid1是table1的外键
  • table1的tid1受到来自table2的tid2的约束
  • table1的tid1必须在table2的tid2中拥有相同的值。所有不符合该约束的结果将不会添加到table1中，同样的，如果在删除或修改table2中的tid2的值的时候，如果table1中还存在与之相同的值，则table2中的值无法被删除或修改
