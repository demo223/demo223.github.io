# 权限修饰符

# 什么是权限修饰符?

用来控制一个成员能够被访问的范围。

# 权限修饰符的分类和范围

权限修饰符：有四种作用范围由小到大（private -> 缺省 -> protected - > public ）

# 权限修饰符的使用

  • 成员变量一般是private
  • 方法一般是public
  • 如果该类只希望本类访问,使用private修饰
  • 如果成员只希望本类以及同一个包下面的其它类和子类访问,使用protected修饰

# final

# final的作用

  • final 关键字是最终的意思，可以修饰（类、方法、变量）
  • 修饰类：表明该类是最终类，不能被继承。
final class Parents {
	public Parents() {
	}
}

class Children extends Parents {
	// 报错:  无法从final 'com.zq.FinalStudy.Parents' 继承
}

# 关于final修饰的变量

final修饰的变量是基本类型：那么变量存储的数据值不能改变。

final修饰的变量是引用类型：那么变量存储的地址值不能改变，但是地址指向的对象内容是可以变化的。

// final修饰的变量是基本数据类型
final int a = 123;
// a = 456;    //  报错
final int b;
b = 222;
// b = 111;    //  报错: final修饰的变量仅能被赋值一次

// final修饰的变量是引用类型
final int[] arr = {1, 2, 3};
arr[0] = 111;
System.out.println(arr[0]); //  111
// 引用类型的变量存储的地址值不能发生改变，但是地址指向的对象内容是可以发生变化的

# 常量

使用了public static final修饰的成员变量，必须有初始化值，而且执行的过程中其值不能被改变。

常量名的命名规范：英文单词全部大写，多个单词下划线连接起来。

public static final double PI = 3.14;
public static final double USER_AGE = 18;

常量做信息配置的原理、优势

  • 在编译阶段会进行“宏替换”：把使用常量的地方全部替换成真实的字面量。
  • 维护系统容易，可读性更好。
