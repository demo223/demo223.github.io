wait()和sleep()的区别

wait()和sleep()的区别

相同点：两者都可以暂停线程

不同点：

  1. 来自不同的类

    wait()来自 Object 类； sleep()来自 Thread 类(Thread的静态方法)

  2. 关于锁的释放

    wait()在等待的过程中会释放锁； sleep()在等待的过程中不会释放锁

  3. 使用的范围

    wait()必须在同步代码块中使用； sleep()可以在任何地方使用

  4. 是否需要捕获异常

    wait()不需要捕获异常； sleep()需要捕获异常；

为什么在Java中，调用wait()方法时不需要捕获InterruptedException异常？

在Java中，调用wait()方法时不需要捕获InterruptedException异常是因为，在调用wait()方法时，如果当前线程被中断了，会抛出一个InterruptedException异常，但是这个异常会被wait()方法内部处理，不会向外抛出。

因此，在Java中，调用wait()方法时不需要捕获InterruptedException异常。

private static final Object lock = new Object();

public static void main(String[] args) throws InterruptedException {
	synchronized (lock) {
		System.out.println("调用wait()方法前");
		lock.wait();  // 等待，直到其他线程调用了lock.notify()或lock.notifyAll()方法
		System.out.println("调用wait()方法后");
	}
}

在Java中，调用sleep()方法需要捕获异常是因为sleep()方法可能会抛出一个InterruptedException异常。

InterruptedException异常是一种检查型异常，它表示当前线程在调用sleep()方法时被中断了。

public static void main(String[] args) {
	System.out.println("调用sleep()方法前");
	try {
		Thread.sleep(1000);  // 暂停当前线程1秒
	} catch (InterruptedException e) {
		// 处理InterruptedException异常
	}
	System.out.println("调用sleep()方法后");
}

public class WaitAndNotifyDemo {
   private static final Object lock = new Object();

   public static void main(String[] args) throws InterruptedException {
      // 创建一个线程，执行同步块中的代码
      new Thread(() -> {
         synchronized (lock) {
            System.out.println("线程1：执行wait()前");
            try {
               lock.wait();  // 线程1进入等待状态
            } catch (InterruptedException e) {
               e.printStackTrace();
            }
            System.out.println("线程1：执行wait()后");
         }
      }).start();

      // 主线程休眠1秒
      Thread.sleep(1000);

      // 创建一个线程，执行同步块中的代码
      new Thread(() -> {
         synchronized (lock) {
            System.out.println("线程2：执行notify()前");
            lock.notify();  // 唤醒等待在lock.wait()处的线程
            System.out.println("线程2：执行notify()后");
         }
      }).start();
   }
}
