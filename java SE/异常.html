# 异常介绍

java中的异常又称为例外，是一个在程序执行期间发生的事件，它中断正在执行程序的正常指令流。为了能够及时有效地处理程序中的运行错误，必须使用异常类，这可以让程序具有极好的容错性且更加健壮。

简单来讲就是

​ 异常是程序在“编译”或者“执行”的过程中可能出现的问题，注意：语法错误不算在异常体系中。

java中发生异常的原因有很多，常见的有

  1. 用户输入了非法数据。
  2. 要打开的文件不存在。
  3. 网络通信是连接中断，或者JVM内存溢出。

# 为什么要使用异常?

  • 异常一旦出现了，如果没有提前处理，程序就会退出JVM虚拟机而终止.
  • 研究异常并且避免异常，然后提前处理异常，体现的是程序的安全, 健壮性。

# 异常层次结构

在java中，Throwable是所有错误（Error）或异常（Exception）的超类，其中，异常又分为编译时异常和运行时异常。

  1. 编译时异常

    RuntimeException以外的异常。这类异常在编译时编译器会提示需要捕获，如果不进行捕获则编译错误。

  2. 运行时异常

    RuntimeException及其之类的异常。这类异常在代码编写的时候不会被编译器所检测出来，是可以不需要被捕获，但是程序员也可以根据需要进行捕获抛出。

从上图可以看到，Throwable类是层次结构中的根类。

请注意，层次结构分为两个分支：错误（Error）和异常（Exception）。

# Error

表示不可恢复的情况，例如Java虚拟机（JVM）内存不足，内存泄漏，堆栈溢出错误，库不兼容，无限递归等。错误通常是程序员无法控制的，我们不应该尝试处理错误。

# Exception

程序可以捕获并处理异常。

当方法内发生异常时，它将创建一个对象。该对象称为异常对象。

它包含有关异常的信息，例如异常的名称和说明以及发生异常时的程序状态。

# 异常分类

异常层次结构还具有两个分支：RuntimeException和IOException。

  1. # RuntimeException(运行时异常)

一个运行时异常发生由于编程错误。它们也称为非检查异常。

这些异常不在编译时检查，而是在运行时检查。一些常见的运行时异常是：

  • API使用不当 - IllegalArgumentException
  • 空指针访问（缺少变量的初始化）- NullPointerException
  • 越界数组访问 ArrayIndexOutOfBoundsException
  • 将数字除以0 ArithmeticException

说白了就是程序员自己代码写的有问题导致的。

  1. IOException(编译时异常)

也称为检查异常。它们由编译器在编译时检查，并提示程序员处理这些异常。

编译时异常是编译阶段就出错的，所以必须处理，否则代码根本无法通过。

# 异常捕获

java的异常处理通过 5 个关键字来实现：try、catch、throw、throws 和 finally。try catch 语句用于捕获并处理异常，finally 语句用于在任何情况下（除特殊情况外）都必须执行的代码，throw 语句用于拋出异常，throws 语句用于声明可能会出现的异常。

try...catch...finally代码块执行的顺序

  • 当try没有捕获到异常时：try语句块中的语句逐一被执行，程序将跳过catch语句块，执行finally语句块和其后的语句；
  • 当try捕获到异常，catch语句块里没有处理此异常的情况：当try语句块里的某条语句出现异常时，而没有处理此异常的catch语句块时，此异常将会抛给JVM处理，finally语句块里的语句还是会被执行，但finally语句块后的语句不会被执行；
  • 当try捕获到异常，catch语句块里有处理此异常的情况：在try语句块中是按照顺序来执行的，当执行到某一条语句出现异常时，程序将跳到catch语句块，并与catch语句块逐一匹配，找到与之对应的处理程序，其他的catch语句块将不会被执行，而try语句块中，出现异常之后的语句也不会被执行，catch语句块执行完后，执行finally语句块里的语句，最后再执行finally语句块后的语句；
  1. try...catch...(finally) (finally可以省略)
try {
  // 代码
} catch (ExceptionType e) { 
  // 捕获块
} finally {
  //finally块
}

  • 将代码放在可能会生成异常的代码放在try块中。

  • 每个try块后面应紧跟着catch 或 finally块。发生异常时，它会被catch紧随其后的块捕获。

  • catch块不能单独使用，必须紧随try块，在一个 try-catch 语句块中可以捕获多个异常类型，并对不同类型的异常做出不同的处理。

public class Test1 {
	public static void main(String[] args) {
		try {
			int a = 10 / 0;
		} catch (NullPointerException e) {
			System.out.println("异常为" + e);
		} catch (ArithmeticException e) {
			System.out.println("异常为" + e);  //  打印
		} finally {
			System.out.println("catch会自动匹配符合条件的异常");
		}
	}
}

如果catch中有return语句，finally还会继续执行吗？

public static int foo(int num) {
	try {
		int a = 10 / 0; //  捕获异常,下面的代码都不会执行
		System.out.println("try中的return");
		num = 1;
		System.out.println("try中num的值");
		return num;
	} catch (Exception e) {
		System.out.println("catch中的return");
		num = 2;
		System.out.println("catch中num的值");
		return num;
     //  这里方法不会结束,依然会执行finally中的语句,return的值会暂时存放到栈里面
	} finally {
		System.out.println("finally中的return");
		num = 3;
		System.out.println("finally中num的值");
		return num;
     //  finally中不建议使用return语句,因为finally中的return语句会覆盖catch中的return结果
	}
}

总结:

  • try中异常后面的代码不会执行
  • 如果try中没有异常，try中return后的结果会保存到栈里面，代码不会终止，会继续执行finally中的语句，然后结束代码
  • catch中的代码，在触发指定的异常后执行，如果后面还存在finally语句，在执行return语句后，return的结果保留在栈中，代码不会结束，会j继续执行finally语句中的代码
  • finally中不建议使用return语句，因为finally中的return语句会覆盖前面的catch中的return结果
  1. try-finally
try {
	// System.exit(0);
	System.out.println("不需要捕获异常的代码");
} finally {
	System.out.println("用于释放资源，在IO流操作和数据库操作中会见到。");
	System.out.println("jvm退出了,finally就不会执行");
}

try-finally可用在不需要捕获异常的代码，可以保证资源在使用后被关闭。例如IO流中执行完相应操作后，关闭相应资源；使用Lock对象保证线程同步，通过finally可以保证锁会被释放；数据库连接代码时，关闭连接操作等等。

try-finally遇见如下情况不会执行

  • 在前面的代码中用了System.exit()退出程序。
  • finally语句块中发生了异常。
  • 程序所在的线程死亡。
  • 关闭CPU。

# 抛出异常

抛出异常有3种方式

  1. 系统自动抛出异常
  2. throw手动抛出异常
  3. throws可能要抛出的异常
public class Test3 {
	public static void main(String[] args) {
		int a = 10;
		int b = 0;
		try {
			System.out.println("系统自动抛出异常");
			int c = a / b;
		} catch (Exception e) {
			System.out.println(e);
			// Exception in thread "main" java.lang.ArithmeticException: / by zero
		}

		try {
			System.out.println("使用throw手动抛出异常");
			if (b == 0) {
				throw new RuntimeException("除数不能为0");
				// java.lang.RuntimeException: 除数不能为)
			}
		} catch (Exception e) {
			System.out.println(e);
		}

		System.out.println("使用throws手动抛出异常");
		try {
			calc(a, b);
		} catch (Exception e) {
			System.out.println(e);
			// java.lang.ArithmeticException: / by zero
		}
		System.out.println("end");
	}

	public static int calc(int a, int b) throws Exception {
		return a / b;
	}
}

throw和throws的区别

  1. throws出现在方法函数头；而throw出现在函数体。
  2. throws表示出现异常的一种可能性，并不一定会发生这些异常；throw则是抛出了异常， 执行throw则一定抛出了某种异常对象。
  3. 两者都是消极处理异常的方式（这里的消极并不是说这种方式不好），都只是抛出或者可能抛出异常，但是不会由函数去处理异常，真正的处理异常由函数的上层调用处理。

# 自定义异常

自定义异常处理类，只包括两部分：

  1. 类的构造方法，构造方法的建立则非常简单，其实只要继承父类（比如 Exception类 ）的构造方法即可。
  2. 重写toString()函数，这是最重要的一点，再toString函数中，描述具体发生了什么错误导致了异常的抛出。这样一来用户才知道究竟是发生了什么错误。
class MyException extends Exception {
	public MyException() {
		super();
	}

	@Override
	public String toString() {
		return "除数不能为0";
	}
}
