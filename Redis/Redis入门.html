# Redis入门

# Redis

# 什么是 Redis ？

  • Redis是一个基于内存的key-value结构数据库。

  • 基于内存存储，读写性能高

  • 适合存储热点数据（热点商品、资讯、新闻）

  • 企业应用广泛

# Redis简介

官网 (opens new window)：

  • Redis是一个开源的内存中的数据结构存储系统，它可以用作 ：数据库、缓存和消息中间件。
  • Redis是用C语言开发的一个开源的高性能键值对(key-value)数据库，官方提供的数据是可以达到100000+的QPS （每秒内查询次数）。它存储的value类型比较丰富，也被称为结构化的NoSql数据库。
  • NoSql（Not Only SQL），不仅仅是SQL，泛指非关系型数据库。NoSql数据库并不是要取代关系型数据库，而是 关系型数据库的补充。

关系型数据库(RDBMS)

  • Mysql
  • Oracle
  • DB2
  • SQLServer

非关系型数据库(NoSql)

  • Redis
  • Mongo db
  • MemCached

# Redis的应用场景

  • 缓存数据
  • 任务队列
  • 消息队列MQ
  • 分布式锁

# Redis下载

Redis安装包分为windows版和Linux版

  • Windows版下载地址：https://github.com/microsoftarchive/redis/releases

  • Linux版下载地址： https://download.redis.io/releases/

# Redis安装

在Window系统安装Redis步骤：

  • 直接解压即可使用

在Linux系统安装Redis步骤：

  1. 将Redis安装包上传到Linux

  2. 解压安装包，命令：

    tar -zxvf redis-4.0.0.tar.gz -C /usr/local
    1

  3. 安装Redis的依赖环境gcc，命令：

    yum install –y gcc-c++
    1

  4. 进入/usr/local/redis-4.0.0，进行编译，命令：

    make
    1

  5. 进入redis的src目录，进行安装，命令：

    make instal
    1

# Redis服务启动与停止

  • Windows系统中启动Redis

    • 双击redis-server.exe即可启动Redis服务
    • 默认端口号为6379，Ctrl + C停止Redis服务

  • Linux系统中启动Redis

    • 使用 ./redis-server
    • 默认端口号为6379，Ctrl + C停止Redis服务

  • 客户端启动

    要想启动当前Redis的客户端，必须先启动当前Redis的服务端

    然后切换至Redis安装目录下，执行src/redis-cli命令

    不论是启动redis-server还是启动redis-cli，实际上就是执行redis-server（或redis-cli）文件，不论在哪里 启动，只要能定位到该文件即可

  • 客户端关闭

    在linux中，可以利用“Redis Shutdown”命令关闭redis服务器，语法为“redis-cli shutdown”；执行失败时返回错误，执行成功时不返回任何信息，服务器和客户端的连接断开，客户端自动退出。

    redis-cli shutdown

# Redis配置文件

Windows中Redis的配置文件：redis.windows.conf

Linux中Redis的配置文件：redis.conf

Redis相关核心配置介绍：

  • Redis后台运行daemonize yes。注意：Windows版不支持
  • Redis密码配置requirepass 123456，默认不开启。
  • Redis默认bind 127.0.0.1只能本地连接，使用#注释掉
  • Redis关闭保护模式protected-mode no，默认开启，开启保护模式时，必须要设置密码才能远程连接

修改Redis配置文件步骤如下：

  1. 在Redis文件目录下执行

    vim redis.conf
    1

  2. 打开文件后搜索关键字，输入/daemonize，找到后进入插入模式把后面的no改为yes

  3. 然后输入

    src/redis-server ./redis.conf
    1

# Redis的外部访问

默认情况下，Redis服务器不允许远程访问，只允许本机访问，所以我们需要设置打开远程访问的功能。

  1. 打开Redis服务器的配置文件 redis.conf

  2. 使用注释符号 # 注释bind 127.0.0.1这行

  3. protected-mode yes 改为 protected-mode no

    注意: protected-mode 是3.2 之后加入的新特性，是为了禁止公网访问redis cache，加强redis安全的。如果你的redis版本低于3.2是找不到该配置项的，所以可以忽略。

  4. 如果还是远程连接不上，注意防火墙要放行6379端口，云服务器去安全组放行端口即可。

  5. 注意启动的时候要指定配置文件的形式启动才能进行远程访问

    在Redis文件目录下执行 src/redis-server ./redis.conf以指定配置文件的规则启动

# Redis的安全问题

# Redis密码设置

  1. 在 redis 目录下 输入src/redis-server ./redis.conf修改配置文件

  2. 找到如下内容，在 requirepass后面设置密码

    #requirepass foobared
requirepass 123456
    1
    2

  3. 登录验证密码是否设置成功

    在 redis 目录下，输入如下命令

     cd /usr/local/redis-4.0.0/
 src/redis-server ./redis.conf
 src/redis-cli
    1
    2
    3

    接下来进行相关的redis操作，如果弹出

    (error) NOAUTH Authentication required.
    1

    则说明密码已经设置成功

    接着输入密码

    auth 密码
    1

    在登录成功后即可进行相关的 redis 操作

  4. 把protected-mode 设置为 yes，外部访问开启密码保护

# 云服务器的安全隐患问题

在没有设置redis 的密码且开启外部访问的时候，很容易收到外部的攻击

外部的用户只需要知道用户的ip地址即可连接它的redis，因此极不安全，所以在开启对外访问的时候一定要开启密码验证。
