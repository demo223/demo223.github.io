# Git常用命令

# git本地仓库

本地仓库常用命令如下：

  • git status 查看文件状态
  • git add 将文件的修改加入暂存区
  • git reset 将暂存区的文件取消暂存或者是切换到指定版本
  • git commit 将暂存区的文件修改提交到版本库
  • git push 推送到远程仓库
  • git log 查看日志

测试环境：本地仓库mycode项目，在项目下创建了一个hello.md文件

# git status

git status 用于查看文件状态

hello.md文件为红色表示改文件没有添加到暂存区

# git add

git add 用于将文件的修改加入暂存区

在执行git add操作后，再次执行git status操作查看可以看到``hello.md`文件已经变成了绿色，表示文件已经加到了暂存区

# git reset

主要有2个功能

  • 将暂存区的文件取消暂存，与上一次的提交(commit)保持一致

    git reset默认取消暂存区的所有文件，也可以使用

    git reset 文件名
    1

    取消暂存区的指定文件

  • 切换到指定版本

    为了演示方便，先将hello.md文件提交到远程仓库，先看下面的几个命令的使用看完再看这个演示

    我们在提交(commit)了hello.md文件后，如果想回退到上次的commit位置，可以执行如下操作

    git reset --hard 版本号
    1

    我们可以通过git log操作来查看日志中的指定版本号

    执行回退命令

    git reset --hard 489f2ac47
    1

    此时查看本地仓库发现hello.md文件已经不存在了

    版本号输入前面几位就可以了，没必要输入整个版本号

# git commit

git commit 命令的作用是将暂存区的文件修改提交到版本库，命令格式：git commit -m '提交的信息'

# git push

git push 用于推送到远程仓库

# git log

git log 用于查看日志

在输入字母Q或q后即可退出日志

# git远程仓库

  • git remote 查看远程仓库
  • git remote add 添加远程仓库
  • git clone 从远程仓库克隆
  • git pull 从远程仓库拉取
  • git push 推送到远程仓库

# git remote

git remote 可以用来查看远程仓库

如果要查看已经配置的远程仓库服务器，可以执行 git remote 命令，它会列出每一个远程服务器的简称。

如果已经克隆了远程仓库，那么至少应该能看到 origin ，这是 Git 克隆的仓库服务器的默认名字。

可以通过-v参数查看远程仓库更加详细的信息

本地仓库配置的远程仓库都需要一个简称，后续在和远程仓库交互时会使用到这个简称

# git remote add

git remote add 添加远程仓库

git remote add 远程仓库地址
1

如果出现报错可以查看git报错日志文章那里的解释

# 分支操作

分支是Git 使用过程中非常重要的概念。使用分支意味着你可以把你的工作从开发主线上分离开来，以免影响开发主线。

本地仓库和远程仓库中都有分支，同一个仓库可以有多个分支，各个分支相互独立，互不干扰。

通过git init 命令创建本地仓库时默认会创建一个master分支。

  • git branch 查看分支
  • git branch [name] 创建分支
  • git checkout [name] 切换分支
  • git push [shortName] [name] 推送至远程仓库分支
  • git merge [name] 合并分支

# 查看分支

git branch 列出所有本地分支

git branch -r 列出所有远程分支

git branch -a 列出所有本地分支和远程分支

# 切换分支

一个仓库中可以有多个分支，切换分支命令格式：

git checkout 分支名称 切换到指定分支

# 推送至远程仓库分支

推送至远程仓库分支命令格式：

git push 远程仓库简称 分支名称 推送至远程仓库指定分支

# 合并分支

合并分支就是将两个分支的文件进行合并处理，命令格式：

git merge 分支名称 将指定分支和当前分支合并

注意：分支合并时需注意合并的方向，在master分支执行操作，结果就是将当前分支合并到master分支。

# 标签操作

Git 中的标签，指的是某个分支某个特定时间点的状态。通过标签，可以很方便的切换到标记时的状态。

比较有代表性的是人们会使用这个功能来标记发布结点（v1.0 、v1.2等）。

  • git tag 查看标签
  • git tag [name] 创建标签
  • git push [shortName] [name] 将标签推送至远程仓库
  • git checkout -b [branch] [name] 检查标签
