# Mybatis动态插入数据

# 存在的问题

为什么要这么做呢？这是为了解决两个问题：

  1. 在插入新记录时，可能只有部分字段有值，而其他字段为空。如果直接将所有字段都添加到 INSERT INTO 语句中，则会使语句变得冗长且难以维护。使用 <trim> 标签可以解决这个问题，只有当字段有值时才将其添加到语句中。
  2. 在插入新记录时，需要保证字段列表和值列表中的字段数量相同，且对应位置的字段名称和值相对应。

如果在 INSERT INTO 语句中添加了所有字段，但是没有传递给语句的参数对象中的值为空，则这些字段在执行插入操作时会被赋值为 NULL。

例如，如果有一个表结构如下：

CREATE TABLE table_name (
  id INT NOT NULL AUTO_INCREMENT,
  name VARCHAR(255) NOT NULL,
  age INT DEFAULT 0,
  PRIMARY KEY (id)
);

则会在表中插入一条新记录，其中 name 字段的值为 'John'，age 字段的值为 NULL。

注意：

  • 如果表中的某个字段有默认值，那么在插入新记录时，如果不为这个字段赋值，则会使用默认值。

如果在 INSERT INTO 语句中添加了所有字段，并且有些字段的值为空，则会使用这些字段的默认值。例如，在上面的表结构中，如果执行如下语句：

INSERT INTO table_name (id, name)
VALUES (1, 'John');
1
2
3

则会在表中插入一条新记录，其中 name 字段的值为 'John'，age 字段的值为默认值 0。

但是，如果表中的某个字段没有默认值，并且设置了 NOT NULL 约束，则在插入新记录时必须为这个字段赋值。否则，就会抛出一个异常，表示无法插入空值。

例如，在上面的表结构中，如果执行如下语句：

INSERT INTO table_name (id)
VALUES (1);
1
2

则会抛出一个异常，因为 name 字段没有默认值，并且设置了 NOT NULL 约束。

# 解决的办法

​ 我们可以使用 <trim> 标签和 <if> 标签可以动态构建 INSERT INTO 语句，只将传入的参数对象中的非空属性添加到语句中。这样可以避免插入无效的空值，并且可以使语句更简洁、易于维护。

例如，如果使用如下的 MyBatis 映射语句：

<insert id="insertRecord" parameterType="Record">
  INSERT INTO table_name
  <trim prefix="(" suffix=")" suffixOverrides=",">
    <if test="id != null">id,</if>
    <if test="name != null">name,</if>
    <if test="age != null">age,</if>
  </trim>
  VALUES
  <trim prefix="(" suffix=")" suffixOverrides=",">
    <if test="id != null">#{id},</if>
    <if test="name != null">#{name},</if>
    <if test="age != null">#{age},</if>
  </trim>
</insert>

注意，使用 <trim> 标签和 <if> 标签动态构建 INSERT INTO 语句时，需要注意以下几点：

  • 使用 prefix 属性指定字段列表和值列表的前缀，使用 suffix 属性指定后缀。
  • 使用 suffixOverrides 属性去掉列表末尾多余的分隔符。
  • 使用 test 属性指定判断条件，只有当条件为真时，才会将字段或值添加到语句中。

例如，有如下的 MyBatis 映射语句：

<insert id="insertRecord" parameterType="Record">
  INSERT INTO table_name
  <trim prefix="(" suffix=")" suffixOverrides=",">
    id,name,age,
  </trim>
  VALUES
  <trim prefix="(" suffix=")" suffixOverrides=",">
    #{id},#{name},#{age},
  </trim>
</insert>

# Mybatis批量插入数据

# 场景一

例如，如果要向 tb_user 表中插入多条记录，可以在 XML 配置文件中定义如下 SQL 语句

<insert id="insertUsers" parameterType="java.util.List">
  INSERT INTO tb_user
  <trim prefix="(" suffix=")" suffixOverrides=",">
    <if test="item.name != null">name,</if>
    <if test="item.age != null">age,</if>
    <if test="item.address != null">address,</if>
  </trim>
  VALUES
  <foreach collection="list" item="item" separator=",">
    <trim prefix="(" suffix=")" suffixOverrides=",">
      <if test="item.name != null">#{item.name},</if>
      <if test="item.age != null">#{item.age},</if>
      <if test="item.address != null">#{item.address},</if>
    </trim>
  </foreach>
</insert>

# 场景二

在场景一的场景下加一个条件，就是批量插入的数据的参数不统一的情况。

如果要插入多条记录，并且每条记录的参数个数也是动态的，则需要在 XML 配置文件中使用 <foreach> 标签遍历每一条记录，并使用 <trim> 和 <if> 标签分别处理字段列表和值列表。

例如，如果要向 tb_user 表中插入多条记录，每条记录的参数个数也是动态的，可以在 XML 配置文件中定义如下 SQL 语句：

<insert id="insertUsers" parameterType="java.util.List">
  INSERT INTO tb_user
  <foreach collection="list" item="item" index="index" separator=",">
    <trim prefix="(" suffix=")" suffixOverrides=",">
      <if test="item.name != null">name,</if>
      <if test="item.age != null">age,</if>
      <if test="item.address != null">address,</if>
  	</trim>
  	VALUES
    <trim prefix="(" suffix=")" suffixOverrides=",">
      <if test="item.name != null">#{item.name},</if>
      <if test="item.age != null">#{item.age},</if>
      <if test="item.address != null">#{item.address},</if>
    </trim>
</insert>

关于上面的 parameterType 属性的解释

注意上面的parameterType 属性指定了传入参数的类型，在这里，parameterType 属性的值为 java.util.List，表示传入的是一个泛型为 User 的 List。所以在 Java 中调用 insert 方法时，需要传入一个泛型为 User 的 List。

如果你希望传入的是一个 User 类型的对象，则可以将 parameterType 设置为 User。例如：

<insert id="insertUser" parameterType="com.example.domain.User">
  insert into user (name, age, address) values (#{name}, #{age}, #{address})
</insert>

但是，如果你希望传入的是一个包含多个 User 类型的对象的 List，则可以将 parameterType 设置为 List。例如：

<insert id="insertUsers" parameterType="java.util.List">
  insert into user
  <foreach collection="list" item="item" index="index" separator=",">
    (name, age, address) values (#{item.name}, #{item.age}, #{item.address})
  </foreach>
</insert>

由于传入的是一个 List，所以在处理参数时，需要使用 item 和 index 属性获取 List 中的每一个元素和索引。例如，在 XML 配置文件中，我们可以使用 #{item.name} 获取 List 中每一个 User 对象的 name 属性。

总之，在使用 <insert> 标签时，parameterType 可以是单个对象类型，也可以是包含多个对象的 List 类型。

这段代码可以避免列数和数据类型不一致的问题。但是如果传入的参数中有空值，那么这条 INSERT 语句可能会出现问题。例如，如果传入的 list 中有一条记录 name 和 age 字段都为空，那么这条记录会被生成出来 (null,null)，这样就会导致语法错误。

解决办法：

思路：

在 service 层中进行判断和过滤掉整条数据全为 null 的数据然后再进行插入操作可以有效避免出现 INSERT INTO 语句报错的情况，并且这样也更加符合程序设计的规范和原则。

但是还存在的一个问题就是当List集合中所有的数据全为null的时候还是会报错，所以我们这里还需要判断List集合的大小，在过滤掉所有数据中全为 null 的数据后，再判断此时List集合的大小。

代码实现

首先，我们可以在 service 层中进行判断和过滤，将 List 集合中整条数据全为 null 的数据进行过滤：

List<User> filteredList = list.stream().filter(item -> item.getName() != null || item.getAge() != null || item.getAddress() != null).collect(Collectors.toList());
1

然后，再判断过滤后的 List 集合的大小，如果大小为 0，说明所有数据都为 null，此时不执行 INSERT INTO 语句；如果大小大于 0，则执行 INSERT INTO 语句。

if (filteredList.size() > 0) {
  // 执行 INSERT INTO 语句
} else {
  // 不执行 INSERT INTO 语句
}

# 其他

实际上使用 SELECT 语句的 UNION ALL 连接多条 SELECT 语句并使用 FROM dual(MySQL中可省略) 语句创建虚拟表的方式也可以实现多行插入的功能。

例如，向表中插入两条记录：

INSERT INTO user(name, birthday) SELECT 'A','19950902' FROM DUAL;
INSERT INTO user(name, birthday) SELECT 'B','19950903' FROM DUAL;
1
2

实际上等同于

INSERT INTO user(name, birthday) VALUES ('A','19950902');
INSERT INTO user(name, birthday) VALUES ('B','19950903');
1
2

这两种方式在效率上并没有明显的差异，可以根据个人喜好选择使用。

需要注意的是，在使用 SELECT 语句的 UNION ALL 连接多条 SELECT 语句时，必须保证每条 SELECT 语句的列数和数据类型与 INSERT INTO 语句中指定的列数和数据类型一致。
