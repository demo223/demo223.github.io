# 线程安全问题

多个线程同时操作同一个共享资源的时候可能会出现业务安全问题，称为线程安全问题。

线程安全问题出现的原因？

  • 存在多线程并发
  • 同时访问共享资源
  • 存在修改共享资源

模拟线程安全问题

有一对夫妻，他们有一个共同的账户，余额是10000元，模拟2人同时去取10000块钱。

代码如下：

创建一个Account账户类

class Account {
	private int money;

	public Account() {
	}
	public Account(int money) {
		this.money = money;
	}

	public void drawMoney(int money) {
		// 由于执行时间太短看不出结果，所以这里让线程休眠1s
		try {
			Thread.sleep(1000);
		} catch (InterruptedException e) {
			throw new RuntimeException(e);
		}
		if (this.money >= money) {
			this.money -= money;
			String name = Thread.currentThread().getName();
			System.out.println(name + "取款" + money);
		} else {
			System.out.println("余额不足");
		}
	}
}

# 线程安全解决方案

如何能保证线程安全呢？

只要让多个线程实现先后依次访问共享资源，这样就解决了安全问题

线程同步的核心思想

加锁，把共享资源进行上锁，每次只能一个线程进入访问完毕以后解锁，然后其他线程才能进来。

同步可以解决线程安全问题的根本原因就在于锁对象上，因此要避免线程安全问题，多个线程必须使用同一个锁对象，否则，不能解决问题。

存在以下3个方法解决线程安全问题

# 同步代码块

对出现问题的核心代码使用synchronized进行加锁

synchronized (锁对象) {
	执行代码块
}
1
2
3

作用：把出现线程安全问题的核心代码给上锁。

原理：每次只能一个线程进入，执行完毕后自动解锁，其他线程才可以进来执行。

  • 规范上：建议使用共享资源作为锁对象。
  • 对于实例方法建议使用this作为锁对象。
  • 对于静态方法建议使用字节码（类名.class）对象作为锁对象

# 同步方法

使用synchronized修饰代码核心方法

修饰符 synchronized 返回值类型 方法名称(形参列表) {
	执行代码块
}

3

作用：把出现线程安全问题的核心方法给上锁。

原理：每次只能一个线程进入，执行完毕以后自动解锁，其他线程才可以进来执行。

同步方法底层原理

同步方法其实底层也是有隐式锁对象的，只是锁的范围是整个方法代码。

  • 如果方法是实例方法：同步方法默认用this作为的锁对象。但是代码要高度面向对象！

  • 如果方法是静态方法：同步方法默认用类名.class作为的锁对象。

是同步代码块好还是同步方法好一点？

同步代码块锁的范围更小，同步方法锁的范围更大

# Lock锁

为了更清晰的表达如何加锁和释放锁，JDK5以后提供了一个新的锁对象Lock，更加灵活、方便。

Lock实现提供比使用synchronized方法和语句可以获得更广泛的锁定操作。

Lock是接口不能直接实例化，这里采用它的实现类ReentrantLock来构建Lock锁对象。

Lock lock = new ReentrantLock();
lock.lock();
	执行代码块
lock.unlock();

# 线程通信

谓线程通信就是线程间相互发送数据，线程间共享一个资源即可实现线程通信。

线程通信的前提：线程通信通常是在多个线程操作同一个共享资源的时候需要进行通信，且要保证线程安全。

在Object.java中，定义了wait()，notify()和notifyAll()等接口。

Object类的等待和唤醒方法

方法 说明
void wait() 让当前线程等待并释放所占的锁资源，
直到哪一个线程调用notify()方法或notifyAll方法
void notify() 唤醒正在等待的线程
void notifyAll() 唤醒正在等待的所有线程
public class Account {
	Account a;	//	账户
	private int money;	//	余额

	public Account() {
	}

	public Account(int money) {
		this.money = money;
	}

	//	模拟取钱操作
	public synchronized void drawMoney(int money) {
		// 这里的this对象表示锁对象
		String name = Thread.currentThread().getName();
		if (this.money >= money) {
			this.money -= money;
			System.out.println(name + "取款" + money + ",余额为" + this.money);
			this.notifyAll();	//	唤醒存钱线程
		} else {
			System.out.println(name + "过来取钱,但是余额不足");
			try {
				this.wait();	//	让当前线程等待并释放锁
			} catch (InterruptedException e) {
				throw new RuntimeException(e);
			}
		}
	}

	//	模拟存钱操作
	public synchronized void saveMoney(int money) {
		String name = Thread.currentThread().getName();
		if (this.money == 0) {
			this.money += money;
			System.out.println(name + "存款" + money + ",余额为" + this.money);
			this.notifyAll();	//	唤醒取钱线程
		} else {
			System.out.println(name + "过来存款,但是里面有钱,不需要存钱");
			try {
				this.wait();	//	让当前线程等待并释放锁
			} catch (InterruptedException e) {
				throw new RuntimeException(e);
			}
		}
	}
}

wait()和sleep()的区别

  1. 来自不同的类

    wait()来自 Object 类； sleep()来自 Thread 类

  2. 关于锁的释放

    wait()在等待的过程中会释放锁； sleep()在等待的过程中不会释放锁

  3. 使用的范围

    wait()必须在同步代码块中使用； sleep()可以在任何地方使用

  4. 是否需要捕获异常

    wait()不需要捕获异常； sleep()需要捕获异常；
