# javaBean

# 什么是javaBean

**JavaBean**是Java中一种特殊的类，可以将多个对象封装到一个对象（bean）中。特点是可序列化，提供无参构造器，提供getter方法和setter方法访问对象的属性。名称中的Bean是用于Java的可重用软件组件的惯用叫法。

Java 之父 James Gosling 在设计 Java 语言，为 Java 组件中封装数据的 Java 类进行命名时，看到了桌子上的咖啡豆，于是灵机一动，就把它命名为JavaBean，bean 翻译为“豆、豆子”。

# javaBean的组成

一般情况下，JavaBean 要满足以下要求：

  • JavaBean 是一个公共类，并提供无参公有的构造方法
  • 需要被序列化并且实现了 Serializable 接口
  • 属性私有化
  • 具有公共的访问属性的 getter 和 setter 方法

# 构造器(构造方法constructor)

定义在类中的，可以用于初始化一个类的对象，并返回对象的地址。

构造函数基本上定义了两个规则。它们分别如下 -

  1. 构造函数名称必须与其类名称相同
  2. 构造函数不能有有显式返回类型

实际上，类的构造方法是有返回值的，当使用 new 关键字来调用构造方法时，构造方法返回该类的实例，可以把这个类的实例当成构造器的返回值，因此构造器的返回值类型总是当前类，无须定义返回值类型。但必须注意不要在构造方法里使用 return 来返回当前类的对象，因为构造方法的返回值是隐式的。

java中的构造方法是一种特殊类型的方法，用于初始化对象。

构造器和方法的区别：

名称 构造器 方法
作用 创建一个类的实例对象 java功能语句
返回类型 没有返回值，没有void 有返回值，或者void
命名 和类名相同；通常为名词，大写开头 通常为动词，小写开头
this 指向同一个类中另外一个构造器，在第一行 指向当前类的一个实例，不能用于静态方法
super 调用父类的构造器，在第一行 调用父类中一个重载的方法
继承 构造器不能被继承 方法可以被继承

注意：

​ 构造方法不能被 static、final、synchronized、abstract 和 native（类似于 abstract）修饰。构造方法用于初始化一个新对象，所以用 static 修饰没有意义。构造方法不能被子类继承，所以用 final 和 abstract 修饰没有意义。多个线程不会同时创建内存地址相同的同一个对象，所以用 synchronized 修饰没有必要。

# 构造器的分类

在Java中，构造函数可以分为3种类型：

  • 无参数构造函数
  • 默认构造函数
  • 参数构造函数
//	无参构造方法
public User() {
}

//	有参构造方法
public User(String name, int age) {
    this.name = name;

# this

this 关键字是java常用的关键字，可用于任何实例方法内指向当前对象，也可指向对其调用当前方法的对象，或者在需要当前类型对象引用时使用。

Java中为解决变量(局部变量和成员变量)的命名冲突和不确定性问题，引入关键字this代表其所在方法的当前对象的引用

注意：

对于 static 修饰的方法而言，可以使用类来直接调用该方法，如果在 static 修饰的方法中使用 this 关键字，则这个关键字就无法指向合适的对象。所以，static 修饰的方法中不能使用 this 引用。并且 Java 语法规定，静态成员不能直接访问非静态成员。

public static void fun(){
	System.out.println("静态方法");
	// this.foo(); //  static修饰的类无法使用this引用
	// this.name;  //  静态成员无法访问非静态成员
}

# this.属性名

当一个类的属性（成员变量）名与访问该属性的方法参数名相同时，则需要使用 this 关键字来访问类中的属性，以区分类的属性和方法中的参数。

//	构造方法
public User(String name, int age) {
    this.name = name;
    this.age = age;
}

//	setter方法
public void setName(String name) {
    this.name = name;
}

//	成员方法
public void getNameAndAge(String name,int age){
    this.name = name;

# this.方法名

this 关键字最大的作用就是让类中一个方法，访问该类里的另一个方法或实例变量。

class User {
	public User() {
	}

	public void foo() {
		System.out.println("foo方法");
	}

	public void bar() {
		this.foo();
		foo();
		new User().foo();
		System.out.println("bar方法");
	}

总结：

引用数据类型的Getter和Setter方法中，不要直接操作原对象，应该返回原对象的一个副本。实际上就是创建一个新的地址，新的堆内存来存储原对象的值。

	System.arraycopy(arr, 0, copyArr, 0, arr.length);
	return copyArr;
}

public void setArr(String[] arr) {

# javaBean代码示例

一个完整的javaBean如下

class People {
    private String name;    //  成员变量
    private int age;    //  成员变量
    private boolean adult;    //  成员变量
    
    //  无参构造方法
    public People() {   
    }

    // 有参构造方法
    public People(String name, int age, boolean adult) {
        this.name = name;
        this.age = age;
        this.adult = adult;
    }

    public String getName() {	    //  成员方法
        return name;
    }

    public void setName(String name) {    //  成员方法
        this.name = name;
    }

    public int getAge() {    //  成员方法
        return age;
    }

    public void setAge(int age) {    //  成员方法
        this.age = age;
    }

    public boolean isAdult() {    //  成员方法
        return adult;
    }

    public void setAdult(boolean adult) {    //  成员方法
        this.adult = adult;
    }
}

# 成员变量和局部变量

由static修饰的变量称为静态变量，其实质上就是一个全局变量。如果某个内容是被所有对象所共享，那么该内容就应该用静态修饰；没有被静态修饰的内容，其实是属于对象的特殊描述。

区别 成员变量(实例变量) 局部变量(类变量)
使用位置 类中、方法外 方法中
初始值 有默认值，无需初始化 没有默认初始值
内存存储位置 堆内存 栈内存
生命周期 随着对象的创建而存在，
随着对象的消 失而消失		 随着方法的调用而存在，
随着方法的运行结束 而消失
使用范围 整个类中 所属的大括号里面
