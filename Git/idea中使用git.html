# 关于.gitignore文件

把要忽略的文件名填进去，Git就会自动忽略这些文件

忽略文件的原则：

  1. 忽略操作系统自动生成的文件，比如缩略图等；
  2. 忽略编译生成的中间文件、可执行文件等，也就是如果一个文件是通过另一个文件自动生成的，那自动生成的文件就没必要放进版本库，比如Java编译产生的.class文件；
  3. 忽略你自己的带有敏感信息的配置文件，比如存放口令的配置文件。

使用规则

  • 一定要放在项目的根目录下
  • 文件名不能更改

设置.gitignore文件

例如，gitee上面在创建仓库的初始化仓库的时候可以选择.gitignore文件，如果是java代码可以选择java

这样在仓库创建后就会在根目录下自动添加一个.gitignore文件

# 本地仓库下文件颜色的含义

还有其他的几种颜色

  • 绿色——已经加入控制暂未提交；

  • 红色——未加入版本控制；

  • 蓝色——加入，已提交，有改动；

  • 白色——加入，已提交，无改动；

  • 灰色——版本控制已忽略文件；

  • 黄色——被git忽略，不跟踪。
