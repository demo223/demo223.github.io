# 多态

# 什么是多态?

它是指在父类中定义的属性和方法被子类继承之后，可以具有不同的数据类型或表现出不同的行为，这使得同一个属性或方法在父类及其各个子类中具有不同的含义。

对面向对象来说，多态分为编译时多态和运行时多态。

  • 编译时多态是静态的，主要是指方法的重载，它是根据参数列表的不同来区分不同的方法。通过编译之后会变成两个不同的方法，在运行时谈不上多态。

  • 运行时多态是动态的，它是通过动态绑定来实现的，也就是大家通常所说的多态性。

# 多态的前提

实现多态有 3 个必要条件：继承、重写和向上转型。只有满足这 3 个条件，开发人员才能够在同一个继承结构中使用统一的逻辑实现代码处理不同的对象，从而执行不同的行为。

  • 继承：在多态中必须存在有继承关系的子类和父类。
  • 重写：子类对父类中某些方法进行重新定义，在调用这些方法时就会调用子类的方法。
  • 向上转型：在多态中需要将子类的引用赋给父类对象，只有这样该引用才既能可以调用父类的方法，又能调用子类的方法。

# 多态的实现

public class Test {
	public static void main(String[] args) {
		Animal a = new Dog();   //  向上转型
		//  调用父类和子类共有的方法cry(),注意这里调用的是覆盖后的cry()方法
		a.fu();   //  父类的方法    (这里的fu方法是父类独有的方法)
		a.cry();  //  汪汪汪       (这里的cry方法在子类中被重写了)
	}
}

class Animal {
	// 子类重写了父类的cry方法
	void cry() {
		System.out.println("动物的叫声");
	}
	// 父类独有的方法
	void fu() {
		System.out.println("父类的方法");
	}
}

class Dog extends Animal {
	@Override
	void cry() {
		System.out.println("汪汪汪");
	}
	// 子类独有的方法
	void zi() {
		System.out.println("子类的方法");
	}
}

# 多态的类型转换

这里所说的对象类型转换，是指存在继承关系的对象，不是任意类型的对象。当对不存在继承关系的对象进行强制类型转换时，java 运行时将抛出 java.lang.ClassCastException 异常。

在继承链中，我们将子类向父类转换称为“向上转型”，将父类向子类转换称为“向下转型”。

向上转型本质: 父类引用指向子类对象。

向下转型本质: 子类对象指向父类引用。

操作 语法
自动类型转换(从子到父,向上转型) Father father = new Son();
强制类型转换(从父到子,向下转型) Son son = (Son) father;
相当于：Son son = (Son) new Son()

注意: 下面的这种向下转型的方式是错误的

Father father = new Father() ---------父类引用指向父类引用
Son son = (Son) father-------------向下转型⬇
// 相当于：Son son = (Son) new Father()

3

此时是子类引用强转父类对象，这是错误的；

转型 特点
向上转型 子类对象转换父类对象的过程，父类引用不能调用子类特有的方法，此时会丢失数据
向下转型 父类对象转换子类对象的过程，是基于向上转型，解决丢失数据

# 向下转型的好处

在上面我们可以看出向上转型后，反而不能使用子类中特有的方法了。

所以向下转型的好处就是可以使用子类特有的方法。

System.out.println("使用向下转型");
Car car1 = new Benz();  //  向上转型
Benz benz1 = (Benz) car1;   //  向下转型
// 相当于
// Benz benz2 = (Benz) new Benz();
// car1.price();    //  报错:父类中不含子类的price方法
benz1.price();  //  正确:可以调用

# instanceof关键字

严格来说 instanceof 是java运算符，由于它是由字母组成的，所以也是 Java 的保留关键字。在 Java 中可以使用 instanceof 关键字判断一个对象是否为一个类（或接口、抽象类、父类）的实例，语法格式如下所示。

boolean result = obj instanceof Class
1

其中，obj 是一个对象，Class 表示一个类或接口。obj 是 class 类（或接口）的实例或者子类实例时，结果 result 返回 true，否则返回 false。

# instanceof关键字的使用

  1. # 声明一个 class 类的对象，判断 obj 是否为 class 类的实例对象(很普遍的一种用法)，如以下代码：

    User user = new User();
System.out.println(user instanceof User);   //  true
System.out.println(user instanceof Object); //  true
Integer a = 123;
System.out.println(a instanceof Integer);   //  true

# 多态案例

  1. 先写一个Usb接口
  2. 写一个Mouse类和keyboard类实现Usb接口
  3. 写一个Computer类,里面添加一个useUsb方法来实现多态
  4. 测试类
public interface Usb {
    public abstract void insert();

    public abstract void pullOut();
}

public class Mouse implements Usb{
	@Override
	public void insert(){
		System.out.println("鼠标的插入方法");
	}

	/**
	 * 子类特有的方法
	 */
	public void click(){
		System.out.println("鼠标的点击方法");
	}

	@Override
	public  void pullOut(){
		System.out.println("鼠标的拔出方法");
	}
}

public class Keyboard implements Usb{
	@Override
	public void insert(){
		System.out.println("键盘的插入方法");
	}

	/**
	 * 子类特有的方法
	 */
	public void input(){
		System.out.println("键盘的打字方法");
	}

	@Override
	public  void pullOut(){
		System.out.println("键盘的拔出方法");
	}
}

public class Computer {
	public void useUsb(Usb u){
		u.insert();
		u.pullOut();

		if (u instanceof Mouse){
			((Mouse) u).click();
		}

		if (u instanceof Keyboard) {
		    ((Keyboard) u).input();
		}
	}
}

public class Test {
	public static void main(String[] args) {
		Computer c = new Computer();
		Mouse mouse = new Mouse();
		Keyboard keyboard = new Keyboard();
		c.useUsb(mouse);
		c.useUsb(keyboard);
	}
}

# 继承与多态的方法优先级问题

根据 Java 多态机制，继承链中对象方法的调用存在一个优先级的问题

  1. this.method(obj) 判断本类中是否存在该方法
  2. super.method(obj) 本类的父类中是否有该方法
  3. this.method((super) obj) 本类中是否有该参数的父类的方法(将obj的父类作为参数的方法)
  4. super.method((super) obj) 本类的父类中是否有该参数的父类的方法

经典例题分析

public class Test {
	public static void main(String[] args) {
		A a1 = new A();	//	A类型对象
		A a2 = new B();	//	向上转型	A类型的B对象
		B b = new B();	//	B类型对象
		C c = new C();	//	C类型对象
		D d = new D();	//	D类型对象
		System.out.println("1--" + a1.show(b));  // 1--A and A
		System.out.println("2--" + a1.show(c));  // 2--A and A
		System.out.println("3--" + a1.show(d));  // 3--A and D

		System.out.println("4--" + a2.show(b));  // 4--B and A

		System.out.println("5--" + a2.show(c));  // 5--B and A
		System.out.println("6--" + a2.show(d));  // 6--A and D
		System.out.println("7--" + b.show(b));   // 7--B and B
		System.out.println("8--" + b.show(c));   // 8--B and B
		System.out.println("9--" + b.show(d));   // 9--A and D
	}
}

例题解析：

  • 第1个例子a1.show(b)：

    a1.show(b)问题根据上面的优先级可以转换为this.show(b)，第一个优先级中A类中不存在show(b)方法，接下来转换为super.show(obj)，当前类A的父类Object类中也不存在show(b)，接下来再看下一个优先级this.show((super)b)发现当前类A中存在show(A obj)方法。 因此输出A and A

    和上面的例子一样的道理可以推出

    a1.show(c)输出A and A

    a1.show(d)输出A and D

  • 第4个例子a2.show(b)

    a2作为对象调用，以a2.show(b)为例，因为是向上转型，那么遵循，编译看左，运行看右的原则，在父类A中没有show(B obj)方法，但同样的有show(A obj),因此要调用该方法；又因为在B中重写了A中的show(A obj),因此在调用的时候就调用了子类重写的show(A obj)，所以最终输出B and A
