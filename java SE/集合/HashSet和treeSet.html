# HashSet和treeSet

# set集合

Set实现了 Collection 接口，它主要有两个常用的实现类：HashSet类和TreeSet类。

Set集合中不能包含重复的对象，并且最多只允许包含一个 null 元素。

对于集合中存入引用数据类型，例如对象，在没有重写对象的hashCode方法的情况下可以存入相同的对象(值相同地址不同)，所以要记得重写hashCode方法。

对于String类型，由于底层已经重写了hashCode，所以直接使用即可。

# HashSet

HashSet 是 Set接口的典型实现，大多数时候使用 Set 集合时就是使用这个实现类。HashSet是按照Hash 算法来存储集合中的元素。因此具有很好的存取和查找性能。

# HashSet特点

  • 不能保证元素的排列顺序，顺序可能与添加顺序不同，顺序也有可能发生变化。
  • HashSet不是同步的，如果多个线程同时访问或修改一个 HashSet，则必须通过代码来保证其同步。
  • 集合元素值可以是null,最多只有一个null元素。

# HashSet常用方法

例如: 初始化一个set集合

HashSet<E> set = new HashSet<>();
1
操作 方法
向集合中添加一个元素 set.add(元素)

检测集合中是否包含指定元素(返回一个布尔值) set.contains(指定元素)
删除集合中的所有元素 set.clear();
检验集合是否为空(返回一个布尔值) set.isEmpty()

注意: 如果向 Set集合中添加两个相同的元素，则后添加的会覆盖前面添加的元素，即在 Set集合中不会出现相同的元素.

# 遍历HashSet

注意:HashSet遍历不能使用普通的for循环遍历，因为它没有索引值

遍历set集合中的元素并存储到数组中

int[] arr = new int[set.size()];
int i = -1;
// set集合不能使用for循环遍历
for (int item : set) {
	i++;
	arr[i] = item;
}
System.out.println(arr);  //  [I@1d251891
System.out.println(Arrays.toString(arr));   //  [1, 2, 3]

# TreeSet

TreeSet类同时实现了 Set接口和 SortedSet接口。SortedSet 接口是Set接口的子接口，可以实现对集合进行自然排序，因此使用 TreeSet类实现的 Set接口默认情况下是自然排序的，这里的自然排序指的是升序排序。

TreeSet只能对实现了 Comparable接口的类对象进行排序，例如Integer，String，Double。

TreeSet<Integer> treeSet = new TreeSet<>();
treeSet.add(56);
treeSet.add(34);
treeSet.add(63);
treeSet.add(29);
// 使用 TreeSet类实现的 Set接口默认情况下是自然排序的
System.out.println(treeSet);    //  [29, 34, 56, 63]

# 将Hashset转换为TreeSet

注意: 如果你想要一个有序的Set，那么最好将元素添加到HashSet然后将其转换为TreeSet ，而不是创建一个TreeSet并向其添加元素。

HashSet<Integer> hashSet = new HashSet<>();
hashSet.add(19);
hashSet.add(31);
hashSet.add(25);
hashSet.add(16);
hashSet.add(34);
hashSet.add(64);
hashSet.add(39);
hashSet.add(27);
// HashSet不能保证元素的排列顺序，顺序可能与添加顺序不同，顺序也可能发生变化
System.out.println(hashSet);    //  [16, 64, 34, 19, 39, 25, 27, 31]
// 如果要排序的话,需要将HashSet转换TreeSet,而不是创建一个TreeSet向其中添加元素
Set<Integer> treeSet = new TreeSet<>(hashSet);
System.out.println(treeSet);    //  [16, 19, 25, 27, 31, 34, 39, 64]

# HashSet和TreeSet的区别

相同点：

  1. HashSet和TreeSet都不包含重复元素，这意味着它们都是不重复的。(可重复加入元素, 但是后面的覆盖前面的元素)
  2. 如果你想要一个有序的Set，那么最好将元素添加到HashSet然后将其转换为TreeSet ，而不是创建一个TreeSet并向其添加元素。

不同点：

  1. HashSet比TreeSet为add，remove，contains，size等操作提供更好的性能（更快）。HashSet提供恒定的时间成本，而对于这样的操作TreeSet提供log(n)时间成本。
  2. HashSet不维护任何元素顺序，而TreeSet元素默认按升序排序。
