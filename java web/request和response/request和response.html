# request和response

  • request：获取请求数据

    • 浏览器会发送HTTP请求到后台服务器[Tomcat]
    • HTTP的请求中会包含很多请求数据[请求行+请求头+请求体]
    • 后台服务器[Tomcat]会对HTTP请求中的数据进行解析并把解析结果存入到一个对象中
    • 所存入的对象即为request对象，所以我们可以从request对象中获取请求的相关参数
    • 获取到数据后就可以继续后续的业务，比如获取用户名和密码就可以实现登录操作的相关业务

  • response：设置响应数据

    • 业务处理完后，后台就需要给前端返回业务处理的结果即响应数据
    • 把响应数据封装到response对象中
    • 后台服务器[Tomcat]会解析response对象,按照[响应行+响应头+响应体]格式拼接结果
    • 浏览器最终解析结果，把内容展示在浏览器给用户浏览

# request请求数据的常用方法

请求行

方法 说明
String getMethod() 获取请求方式
String getContextPath 获取虚拟目录(项目访问路径)
StringBuffer getRequestURL 获取URL(统一资源定位符)
StringBuffer getRequestURI 获取URI(统一资源标识符)

请求地址：http://localhost:8080/requestDemo01?username=zs&password=123

//1. getMethod()：获取请求方式
System.out.println(request.getMethod());    //  GET
//2. getContextPath()：获取虚拟目录
System.out.println(request.getContextPath());   //   (xml的path标签里面设置了/)
//3. getRequestURL()：获取请求路径URL
System.out.println(request.getRequestURL());    //  http://localhost:8080/requestDemo01
//4. getRequestURI()：获取请求路径URI
System.out.println(request.getRequestURI());    //  /requestDemo01
//5. getQueryString()：获取请求参数
System.out.println(request.getQueryString());   //  username=zs&password=123

# 请求转发

请求转发：在服务器内部的一种资源跳转方式

请求转发特点：

  • 浏览器地址栏路径不发生变化
  • 只能转发到当前服务器的内部资源
  • 一次请求，可以在转发的资源间使用request共享数据

实现方式

req.getRequestDispatcher("资源B路径").forward(request,response);
1

只能在当前服务器(项目)内部资源进行跳转，不能跳转到外部路径。

# 数据共享

在请求转发的过程中，2个资源之间可以进行数据共享，可以使用Request域对象。

Request对象的方法

方法 说明
void setAttribute(String name，object key) 存储数据到request域中
Object getAttribute(Sting name) 根据key获取value
void removeAttribute(String name) 根据key，删除request域中的键值对

浏览器在访问/request2路径的时候，在request1资源中可以添加数据到request域对象中

request.setAttribute("username","张三");
request.setAttribute("age",26);
request.getRequestDispatcher("/request2").forward(request, response);

# 响应数据在浏览器乱码问题

例如，谷歌浏览器默认网页是GBK的编码格式，而通过Response获取的字符输出流默认编码ISO-8859-1

解决办法：

response.setContentType("text/html;charset=utf-8");
1

对于request和response出现的乱码问题，我们可以在代码一开始就加入如下代码，防止出现乱码。

request.setCharacterEncoding("utf-8");
response.setContentType("text/html;charset=utf-8");

# 重定向

重定向：一种资源跳转方式

重定向特点：

  • 浏览器地址栏路径发生变化
  • 可以重定向到任意位置的资源（服务器内部、外部均可）
  • 两次请求，不能在多个资源使用request共享数据

重定向的实现方式

response.sendRedirect(request.getContextPath() + "/跳转路径");
1

重定向的跳转路径可以是任意位置，服务器和外网都可以。

注意：

​ 重定向需要加虚拟目录，也就是getContextPath()；请求转发不需要加虚拟目录，其他跳转都需要加虚拟目录。例如<a href = '路径'>，<from action = "路径">等都需要加上虚拟目录。

# 关于http中转发和重定向的区别

  1. 重定向 2 次请求，请求转发 1 次请求
  2. 重定向地址栏会变，请求转发地址栏不变
  3. 重定向是浏览器跳转，请求转发是服务器跳转
  4. 重定向可以跳转到任意网址，请求转发只能跳转当前项目
  5. 重定向需要加虚拟目录，请求转发不需要加虚拟目录

# Response 响应字节数据

例如，文件下载

浏览器在访问下载资源的网址后自动下载文件

在响应数据之前告诉浏览器以附件的方式解析响应体中的内容即可

response.setHeader("content-disposition", "attachment;filename=文件名称");
1

需求：将img目录下中的某个图片读取到程序中，发送给客户端浏览器。

// 设置访问下载图片到本地
response.setHeader("Content-Disposition", "attachment;filename=abc.jpg");
ServletOutputStream sos = response.getOutputStream();
FileInputStream fis = new FileInputStream("src/main/webapp/img/a.jpg");
// 复制输入流文件到输出流
IOUtils.copy(fis, sos);
