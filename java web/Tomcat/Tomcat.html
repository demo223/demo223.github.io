# Tomcat

Tomcat是Apache的一个核心项目，免费开源的轻量级Web服务器，支持Servlet/JSP少量JavaEE规范。 Tomcat也被称为Web容器、Servlet容器。

# 安装Tomcat

# 使用Tomcat

  1. 修改Tomcat的默认启动端口，打开conf/server.xml文件，修改port端口号

     <Connector port="8080" protocol="HTTP/1.1"

  2. 启动Tomcat，有如下2种方式

    • 直接 win + R 然后输入 cmd 回车，输入startup启动Tomacat
    • 进入到 bin目录下，双击 startup.bat来启动Tomcat

  3. 验证Tomcat是否启动

    在启动Tomcat后，在浏览器的地址栏输入 localhost:8080，如果看到Apache Tomcat的界面就证明启动成功了。

  4. 启动过程中出现的问题

    1. 出现界面一闪而过的，检查 JAVA_HOME 环境变量是否正确

    2. 出现端口占用问题，找到当前占用的端口将其关掉。

      例如检查8080端口的占用问题 netstat -ano|findstr "8080"，执行后出现如下结果

        TCP    0.0.0.0:8080           0.0.0.0:0              LISTENING       20192
  TCP    [::]:8080              [::]:0                 LISTENING       20192
      1
      2

      关闭8080端口对应的服务 taskkill -F -PID 20192，然后再启动Tomcat即可解决。

    3. 启动界面出现乱码问题，将UTF-8改为GBK即可。

# 在idea中创建Maven Web项目

# 方法1：使用Maven骨架创建

将本地Tomcat 集成到 IDEA 中，然后进行项目部署即可

在idea中选择Maven Archetype通过骨架方式创建项目

创建完成后删除 pom.xml文件中一些不要的信息。留下的pom.xml代码如下

<?xml version="1.0" encoding="UTF-8"?>

<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <groupId>org.example</groupId>
    <artifactId>tomcat-test</artifactId>
    <version>1.0-SNAPSHOT</version>
    <!--  打包方式 默认为jar包，这里选择war包-->
    <packaging>war</packaging>
</project>

xxx.war和 xxx.war exploded这两种部署项目模式的区别?

  • war模式部署成功后，Tomcat的webapps目录下会有部署的项目内容
  • war exploded模式部署成功后，Tomcat的webapps目录下没有，而使用的是项目的target目录下的内容进行部署
  • 建议选war模式进行部署，更符合项目部署的实际情况

# 方法2：使用JBLJavaToWeb插件

建议使用下面这种方法来创建web项目

在idea中安装JBLJavaToWeb插件，然后创建一个普通的MAV了项目然后右键项目选择JBLJavaToWeb即可将Maven项目转换为web项目。

还可以借助tomcat maven插件来启动Tomcat服务器，在pom.xml的project标签中添加tomcat插件即可

<build>
    <plugins>
        <!--Tomcat插件 -->
        <plugin>
            <groupId>org.apache.tomcat.maven</groupId>
            <artifactId>tomcat7-maven-plugin</artifactId>
            <version>2.2</version>
            <configuration>
                <port>8080</port>
                <uriEncoding>utf-8</uriEncoding>
                <path>/</path>
            </configuration>
        </plugin>
    </plugins>
</build>

上面的代码configuration标签中的port配置的是tomcat启动的端口号，path配置的是项目的访问路径，/表示项目的根路径，后续在浏览器访问的时候就不需要再写项目名称到路径中了。

例如：未配置访问路径: http://localhost:80/tomcat-demo2/a.html 配置后访问路径: http://localhost:80/a.html 如果配置成 /hello,访问路径会变成什么？答案: http://localhost:80/hello/a.html

uriEncoding设置为utf-8是为了解决GET请求中文参数乱码问题。

# 生命周期

  • 生命周期：对象的生命周期指一个对象从被创建到被销毁的整个过程。

  1. init 方法：在Servlet实例化之后，容器将调用Servlet的init方法初始化这个对象，该方法只调用一次

  2. service 方法：每次访问Servlet时，都会调用service方法对请求进行处理

  3. distory 方法：当服务器正常关闭时，就会调用destroy方法完成资源的释放

上面的代码中当我们第一次访问服务器中的地址的时候，会执行init方法和service方法，这里也可以调成在启动Tomcat服务器的时候就执行init方法

@WebServlet(value = "/start",loadOnStartup = 10)

loadOnStartup设置成任意一个整数数即可

  1. Servlet生命周期中涉及到的三个方法，这三个方法是什么?什么时候被调用?调用几次?

涉及到三个方法，分别是 init()、service()、destroy()

init方法在Servlet对象被创建的时候执行，只执行1次

service方法在Servlet被访问的时候调用，每访问1次就调用1次

destroy方法在Servlet对象被销毁的时候调用，只执行1次
