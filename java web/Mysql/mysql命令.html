# SQL分类

SQL语句，根据其功能，主要分为四类：DDL、DML、DQL、DCL。

类型 说明
DDL (Data Definition Language) 数据定义语言，用来定义数据库对象(数据库，表， 字段)
DML (Data Manipulation Language) 数据操作语言，用来对数据库表中的数据进行增删改
DQL (Data Query Language ) 数据查询语言，用来查询数据库中表的记录
DCL (Data Control Language) 数据控制语言，用来创建数据库用户、控制数据库的 访问权限

# 数据库操作命令

  1. 进入mysql

    mysql -u root -p
    1

  2. 查询所有数据库

    show databases;
    1

  3. 创建数据库(数据库如果已存在会报错：提示该数据库已存在)

    create database 数据库名称;
    1

  4. 如果数据库不存在则创建数据库

    create database if not exists 数据库名称;
    1

  5. 创建数据库并指定字符集

    create database 数据库名称 default charset utf8mb4;
    1

  6. 删除数据库(数据库如果不存在会报错：提示该数据库不存在)

    drop database 数据库名称;
    1

  7. 如果数据库存在则删除数据库

    drop database if exists 数据库名称;
    1

  8. 进入(切换)指定数据库

    use 数据库名称;
    1

  9. 查看当前操作数据

    select database();
    1

# 数据类型

MySQL中的数据类型有很多，主要分为三类：数值类型、字符串类型、日期时间类型。

# 数值类型

类型 大小 范围（有符号signed） 范围（无符号unsigned） 用途
tinyint 1 Bytes (-128，127) (0，255) 小整数值
smallint 2 Bytes (-32 768，32 767) (0，65 535) 大整数值
mediumint 3 Bytes (-8 388 608，8 388 607) (0，16 777 215) 大整数值
int或integer 4 Bytes (-2 147 483 648，2 147 483 647) (0，4 294 967 295) 大整数值
bigint 8 Bytes (-2^63，2^63-1) (0，2^64-1) 极大整数值
float 4 Bytes (-3.40 E+38， 3.40 E+38) 0，(1.17E-38，3.40E+38) 单精度 浮点数值
double 8 Bytes (-1.79E+308， 1.79E+308) 0，(2.22E-308，1.79E+308) 双精度 浮点数值
decimal 依赖于M(精度)和D(标度)的值 依赖于M和D的值 小数值

  • 例如年龄字段(特定的大小，不能是负数)

    age tinyint unsigned
    1

  • 例如考试分数字段(特定的大小假设为100分，不能为负数，最多1位小数)

    score double(4,1)
    1

# 字符串类型

类型 大小 用途
char 0-255 bytes 定长字符串
varchar 0-65535 bytes 变长字符串
tinyblob 0-255 bytes 不超过 255 个字符的二进制字符串
tinytext 0-255 bytes 短文本字符串
blob 0-65 535 bytes 二进制形式的长文本数据
text 0-65 535 bytes 长文本数据
mediumblob 0-16 777 215 bytes 二进制形式的中等长度文本数据
mediumtext 0-16 777 215 bytes 中等长度文本数据
longblob 0-4 294 967 295 bytes 二进制形式的极大文本数据
longtext 0-4 294 967 295 bytes 极大文本数据

char 与 varchar 都可以描述字符串，char是定长字符串，指定长度多长，就占用多少个字符，和 字段值的长度无关 。而varchar是变长字符串，指定的长度为最大占用长度 。相对来说，char的性能会更高些。

  • 例如用户名字段(长度不一，假设最长不超过50位)

    username varchar(50)
    1

  • 例如性别(要么是男，要么是女)

    gender char(1)
    1

  • 例如手机号默认是11位

    phone char(11)
    1

设计一张员工信息表，要求如下：

  1. 编号（纯数字）
  2. 员工工号 (字符串类型，长度不超过10位)
  3. 员工姓名（字符串类型，长度不超过10位）
  4. 性别（男/女，存储一个汉字）
  5. 年龄（正常人年龄，不可能存储负数）
  6. 身份证号（二代身份证号均为18位，身份证中有X这样的字符）
  7. 入职时间（取值年月日即可）

		sex CHAR ( 1 ) COMMENT '性别',
		age TINYINT UNSIGNED COMMENT '年龄',

# 修改表

  1. 添加字段

    alter table 表名 add 字段名 数据类型(长度) [comment 注释] [约束];
    1

    例如向t_test表中添加一个昵称字段nickname

    alter table t_test add nickname varchar(20) comment '昵称';
    1

  2. 修改数据类型

    alter table 表名 modify 字段名 修改后的数据类型(长度);
    1

  3. 修改字段名和字段类型

    alter table 表名 change 旧字段名 新字段名 类型(长度) [comment '注释'] [约束];
    1

    例如修改t_test表中的nickname字段

    alter table t_test change nickname nickName char(3) comment '昵称';
    1

  4. 删除字段

    alter table 表名 drop 字段名;
    1

  5. 修改表名

    alter table 旧表名 rename to 新表名;
    1
