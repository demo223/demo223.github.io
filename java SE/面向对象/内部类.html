# 内部类

# 内部类介绍

Java 一个类中可以嵌套另外一个类就是内部类

广泛意义上的内部类一般来说包括这四种：

  • 静态内部类

  • 非静态内部类

    • 成员内部类

    • 局部内部类

    • 匿名内部类

# 静态内部类

静态内部类可以使用 static 关键字定义，静态内部类我们不需要创建外部类来访问，可以直接访问它。

class Outer {    //  外部类
	static class Inner1 {   //  静态内部类
		public Inner1() {
			System.out.println("静态内部类");
		}
	}
}

# 成员内部类

非静态内部类是一个类中嵌套着另外一个类。 它有访问外部类成员的权限， 通常被称为内部类。

由于内部类嵌套在外部类中，因此必须首先实例化外部类，然后创建内部类的对象来实现。

class Outer {    //  外部类
	class Inner2 {    //  非静态内部类
		public Inner2() {
			System.out.println("非静态内部类");
		}
	}
}

# 成员内部类的应用

如果我们的内部类不想轻易被任何人访问，可以选择使用private修饰内部类，这样我们就无法通过创建对象的方法来访问，想要访问只需要在外部类中定义一个public修饰的方法，间接调用。这样做的好处就是，我们可以在这个public方法中增加一些判断语句，起到数据安全的作用。

public class Outer {
	public static void main(String[] args) {
		Outer outer = new Outer();
		Outer.Inner inner1 = outer.getInner("管理员");
		Outer.Inner inner2 = outer.getInner("张三");
		if (inner1 != null) {
		    inner1.show();
		}
		if (inner2 != null) {
			inner2.show();
		}
		// outer.getInner("张三");
	}

	public Inner getInner(String user) {
		if (user.equals("管理员")) {
			return new Inner();
		} else {
			System.out.println("无权限访问");
			return null;
		}
	}

	private class Inner {
		public void show() {
			System.out.println("展示机密文件");
		}
	}
}

# 局部内部类(了解)

定义在一个方法或者一个作用域里面的类，它和成员内部类的区别在于局部内部类的访问仅限于方法内或者该作用域内。

public class Outer {
	public void test() {
		System.out.println("Outer类中的方法");
		class Inner {
			public void show() {
				System.out.println("局部内部类的使用(只能在方法中使用)");
			}
		}
		Inner inner = new Inner();
		inner.show();
	}

	public static void main(String[] args) {
		new Outer().test();
	}
}

# 匿名内部类(重点)

匿名内部类本质上是一个没有名字的局部内部类。

正因为没有名字，所以匿名内部类只能使用一次，它通常用来简化代码编写

但使用匿名内部类还有个前提条件：必须继承一个父类或实现一个接口

new 类|抽象类名|或者接口名() {
	重写方法;
};

# 抽象类的基础上实现匿名内部类

例如：有一个如下的抽象Animal类，我们想实现子类dog重写后的抽象方法eat，在不使用匿名内部类时

abstract class Animal {
   public abstract void eat();
}
class Dog extends Animal {
   @Override
   public void eat() {
      System.out.println("子类重写父类方法");
   }
}

public class Test {
	public static void main(String[] args) {
		Animal a = new Dog();
		a.eat();    //  子类重写父类方法
	}
}

		// Animal b = new Animal();
		Animal b = new Animal(){
			@Override

				System.out.println("子类重写父类方法");
			}
		};
		b.eat();    //  子类重写父类方法

}

# 在接口上使用匿名内部类

例如有一个Usb接口，我们要实现接口中的方法

# 在不使用匿名匿名内部类的情况下
interface Usb {
	void insert();
}

class Computer implements Usb {
	@Override
	public void insert() {
		System.out.println("电脑插入USB接口");
	}
}

public class Test {
	public static void main(String[] args) {
        Computer computer = new Computer();
        computer.insert();  //  电脑插入USB接口
	}
}

# 匿名内部类的特点

  • 匿名内部类没有名字

  • 匿名内部类只能使用一次

  • 匿名内部类就是内部类的简化写法

  • 匿名内部类是一个没有名字的内部类，同时也代表一个对象。

  • 匿名内部类的对象类型，相当于是当前new的那个类型的子类类型

# 内部类的使用场景和好处

匿名内部类可以作为一个对象，直接传输给方法。

interface IRun {
	void running();
}

public class Test {
	public static void main(String[] args) {
		allRun(new IRun() {
			@Override
			public void running() {
				System.out.println("乌龟在跑步");
			}
		});

		allRun(new IRun() {
			@Override
			public void running() {
				System.out.println("兔子在跑步");
			}
		});

		allRun(new IRun() {
			@Override
			public void running() {
				System.out.println("老鹰在飞");
			}
		});
	}

	public static void allRun(IRun ir) {
		ir.running();
	}
}

# Thread类的匿名内部类实现

public class Test {
	public static void main(String[] args) {
		Thread thread = new Thread() {
			public void run() {
				for (int i = 0; i < 10; i++) {
					System.out.print(i + " ");
				}
			}
		};
		thread.start();
	}
}
